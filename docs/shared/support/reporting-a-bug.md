# Signaler un bug

## Objectif

Signaler un comportement anormal de la plateforme avec les bonnes informations pour qu'il soit reproduit, diagnostiqué et corrigé rapidement.

## Quand utiliser cette page

- Quand une action ne produit pas le résultat attendu
- Quand une page affiche une erreur ou se charge anormalement lentement
- Quand un membre signale un problème que vous ne savez pas expliquer

## Ce qui aide vraiment

Un bug bien décrit est un bug rapidement corrigé. Indiquez systématiquement :

1. **Le contexte** : qui (rôle), quoi (action), où (URL ou écran)
2. **Les étapes pour reproduire** : 1. clic ici, 2. saisie de…, 3. clic là
3. **Le comportement observé** vs **le comportement attendu**
4. **La capture d'écran** ou la vidéo (préférable)
5. **L'environnement** : navigateur, version, OS, appareil
6. **La date et l'heure** approximatives du problème (utile pour les logs)

## Résultat attendu

Un signalement actionnable, traité selon la criticité (bloquant / gênant / mineur).

## Bonnes pratiques

!!! tip "Reproduire avant de signaler"
    Avant d'envoyer le signalement, essayez de reproduire le bug une seconde fois en session privée (Chrome → fenêtre privée). Cela élimine 80% des faux positifs liés au cache navigateur.

## Points d'attention

!!! warning "Bug bloquant ou non ?"
    Si le bug empêche l'accès au réseau ou compromet des données, écrivez directement à `support@terrilink.io` avec la mention **[URGENT]** dans le sujet. Pour les bugs mineurs ou cosmétiques, le formulaire intégré suffit.
